面试准备

发表于 2021-03-09 更新于 2021-05-16 分类于 面试

面试准备

C++基础

C++ 面向对象三大特性

  1. 封装
  2. 继承
  3. 多态

多态

多态就是不同继承类的对象根据同一消息做出不同的反应，基类的指针指向或者绑定到子类的对象，使得基类的指针呈现不同的表现方式。

虚函数的机制

虚函数是必须在class里面声明的函数，以 virtual 关键字声明。

所有虚函数的地址都会放在所属的类的虚函数表 V-table 中，基类中声明的虚函数，继承到子类中依然是虚函数，也会放在子类的虚函数表中。虚函数表的指针存在对象实例的最前。

虚函数的创建：

基类：首先在基类声明中找到所有的虚函数，按照其声明顺序，编码0,1,2,3,4……，然后按照此声明顺序为基类创建一个虚函数表，其内容就是指向这些虚函数的函数指针，按照虚函数声明的顺序将这些虚函数的地址填入虚函数表中

子类：首先将基类的虚函数表复制到该子类的虚函数表中。若子类重写了基类的虚函数show，则将子类的虚函数表中存放show的函数地址(未重写前存放的是基类的show虚函数的函数地址)更新为重写后函数的函数指针。若子类增加了一些虚函数的声明，则将这些虚函数的地址加到该类虚函数表的后面

虚函数表的访问：

  1. 首先判断指针所指的函数是否是虚函数，是则采用动态联编（使用虚函数表确定调用的函数），不是则采用静态编译（由调用指针的类型决定调用哪个类的成员函数）
  2. 如果是虚函数，则访问指向对象的虚函数表。
  3. 找到函数在基类虚函数表中的位序
  4. 在对象类的虚函数表中找到同样位序的函数，调用此函数

虚函数表的建立时间：编译阶段，将虚函数的地址放在虚函数表中。虚表指针保存的位置在对象内存空间最前面的位置。

虚函数表与类绑定，所有对象共享一个虚函数表，对象首地址存放的是指向虚函数表的指针

重载： 同一个类中，函数名相同，参数和返回值类型可以不同

重写： 子类重写父类的虚函数，参数和返回类型必须相同，不能有static

隐藏： 子类隐藏父类的函数，只要是同名函数，父类函数都会被隐藏

构造函数一般不声明为虚函数，因为指向虚函数表的指针只有在对象生成后才有

析构函数一般声明为虚函数，防止内存泄漏。当一个基类的指针绑定或者指向一个子类对象，如果析构函数不声明为虚函数，就会调用基类函数的析构函数，派生类特有的成员无法释放，导致内存泄漏。

栈和堆

32位系统中，一个进程4GB的虚拟内存空间，128M的不可访问区，1G的内核空间，3G的用户空间。

区别：

  1. 管理方式不同，堆空间由程序员用new和delete申请和释放，栈由编译器在需要的时候分配，不需要的时候自动清楚
  2. 空间大小不同，Linux下栈的默认大小为8MB，堆的大小理论上来说可以占满整个用户空间
  3. 栈由高地址向低地址扩展。堆由低地址向高地址扩展
  4. 只要栈的剩余空间大于程序申请的空间就可以分配，否则异常提示栈溢出
  5. 操作系统有记录空闲内存地址的链表，系统收到程序的申请时遍历该链表，找到第一个满足申请空间的堆节点，分配空间
  6. 频繁的new和delete会产生内存碎片

全局变量定义在头文件中会出现重定义错误，每一个包含该头文件的源文件都会重新定义这个变量，extern关键字解决，extern是声明，不是定义

static把这个变量的作用域限制在此文件内，如果有其他文件引用此变量会产生引用错误

静态局部变量只有在程序结束时才会被销毁

.h文件中一般只放函数声明，防止编译阶段出现重定位错误

如何限制类的对象只能在堆上创建

  1. 将析构函数设为私有，但会导致无法解决继承问题，析构函数一般是虚函数
  2. 构造函数设置为 protected 并提供一个 public的静态函数来完成构造而不是在类的外部使用 new构造。同时将析构函数也设为 protected

限制类的对象只能在栈上创建

将 operator new() 函数设为私有

类

空类占一字节，编译时自动生成6个成员函数：默认构造函数，析构函数，拷贝函数，赋值运算符，取址运算符，取址运算符常量版本

拷贝构造函数必须为引用是为了防止无限次的递归调用构造函数造成栈溢出

C++ 类对象的初始化顺序

-> 基类的构造函数

->派生类成员变量的构造函数

-> 派生类自己的构造函数

析构顺序与构造顺序相反

为什么用成员初始化列表会块一些

如果不用成员初始化列表，在类的构造函数之前成员变量调用默认构造函数初始化，之后在类的构造函数体中调用成员变量的构造函数。成员函数列表直接调用了成员函数对应的构造函数，减少了默认构造函数的调用。

内存

深拷贝与浅拷贝的区别

深拷贝：该对象和原对象占用不同的内存空间

浅拷贝：对象与原对象占用同一块内存空间

memcpy函数

实现：

提高效率，对齐的地址按cpu字长拷贝，不对齐的时候按照一个字节一个字节拷贝

new 与 malloc 的区别

malloc free 都是库函数，new 和 delete都是关键字

new 申请空间时，无需指定分配空间的大小，malloc需要指明空间的大小

new 分配失败会抛出bad-alloc 异常，malloc 会返回空指针

delete 首先执行该对象的析构函数，之后调用 operator delete 来释放所占的内存空间

内存泄漏

由于疏忽或者错误导致程序未能释放已经不再使用的内存

丢失了对该内存的控制

如何防止内存泄漏

  1. 将内存分配和内存释放封装在类中
  2. 使用智能指针

智能指针

  1. shared_ptr（共享指针）：资源可被多个指针共享。通过use_count() 查看资源所有者的个数，release() 释放所有权。
  2. unique_ptr （独占指针）：资源只能被一个指针占有，不能拷贝构造和赋值，可以进行移动构造和移动构造赋值，调用 std::move() 函数
  3. weak_ptr （弱指针）：指向 共享指针指向的对象，解决 shared_ptr 带来的循环引用问题

实现方式：计数原理

  • template头
  • 两个成员变量，一个 T _ptr, 一个计数 size_t _count
  • 构造函数，默认参数为 nullptr
  • 析构函数，计数原理，为零则 delete
  • 拷贝构造函数，参与为shared_ptr的引用，更改_ptr 和 _count 的指向，(*this->$_$count)++。
  • 赋值运算符，返回 shared_ptr 的引用，首先判断两个资源是否相等，如果 this->_ptr 存在，释放资源。然后将成员变量改为对方的成员变量，计数加一
  • 指针运算符，返回资源引用
  • -> 运算符，返回资源指针

模板

什么是模板

创建类或者函数的蓝图或者公式，分为函数模板和类模板

函数模板与类模板的区别

函数模板会根据函数的实参来实例化，类模板实例化要在程序中显式指定

函数模板只能全特化，类模板可以全特化，也可以偏特化

什么是可变参数模板

接受可变数目参数的模板函数或者模板类，将可变数目的参数成为参数包，包括模板参数包，函数参数包。

什么是模板特化，为什么特化？

因为在某些特定类型的实参，模板并非都合适，并非都能实例化，

模板在某种特定类型下的具体实现，分为函数模板特化，类模板特化

全特化：所有模板参数都实例化

偏特化：部分参数实例化

静态链接与动态链接

静态链接：在编译阶段链接，要把调用的函数或者过程链接到可执行文件中，成为可执行文件的一部分，lib 包含在exe文件中。

动态链接：

  1. 装载时动态链接：编译前明确知道要调用DLL中的哪几个函数，编译时在目标函数中只保留必要的链接信息，而不保留DLL函数的代码。程序执行时，调用函数的时候利用链接信息加载DLL函数代码，主要为了代码共享，全部函数加载进内存
  2. 运行时动态链接，编译之前并不知道会调用哪些DLL函数，在运行过程中根据需要决定调用哪个函数，将调用的函数加载进内存

动态链接更省内存，并且更新更快。

STL

vector 与 list 的区别

vector：连续存储的容器，底层由动态数组实现，如果没有空间会扩容为原来空间的两倍，支持随机访问。如果需要随机访问，不在乎插入与删除的效率，使用 vector

list：不连续的，底层由双向链表实现，不支持随机访问，每插入一个元素都会分配空间，每删除一个元素都会释放空间。如果需要高效的插入与删除，使用list

set 与 map 与 unordered_map的区别

set：set中的元素只是关键字的简单集合，不允许有重复，不支持下标操作，底层也是红黑树。

map：底层由红黑树实现，内部实现了排序，元素是键值对，支持下标操作,根据键值排序

unordered_map：底层由hashtable实现，内部不排序。哈希冲突一般使用 开放寻址法和拉链法

AVL树与红黑树

AVL树：平衡二叉搜索树，每个节点的左右子树的高度差不超过1，插入会破坏平衡，这时候需要旋转，外侧插入单旋转，内测插入双旋转

红黑树：每个节点不是红色就是黑色，根节点为黑色，红节点的子节点必须为黑，任一节点到NULL的任何路径，所经过的黑节点数相同。插入的节点必须是红节点

stack 的底层实现：deque，封闭头端开口

MySQL

数据库引擎

InnoDB：默认事务隔离级是可重复读，

NDB cluster：事务不满足持久性和隔离性

Memory，MyIsam不支持事务

数据库三范式

第一范式：对属性的原子性，要求属性有原子性，不可再分解

第二范式：对记录的唯一性，在第一范式的基础上，要求所有非主键属性完全依赖于主键，不存在部分依赖

第三范式：对字段的冗余性，在第二范式的基础上，任何字段不能由其他字段派生出来，不存在传递依赖

范式化设计与反范式化设计的优缺点

范式化设计：

  1. 优点：可以减少数据冗余，数据表更新快，体积小
  2. 缺点：查询需要关联多个表，导致性能降低，更难进行索引优化

反范式化设计：

  1. 优点：可以减少表的关联，可以更好的进行索引优化
  2. 缺点：存在数据冗余以及数据维护异常，对数据的修改需要更多的成本

事务特性

事务（Transaction）是访问和更新数据库的程序执行单元；事务中可能包含一个或多个sql语句，这些语句要么都执行，要么都不执行。

Atomic：原子性，事务的操作要么都做要么都不做，实现基于 undo log

Consistency：一致性，事务执行结束后，数据库的完整性约束没有被破坏，事务执行前后都是合法的数据状态

Isolation：隔离性，保证事务执行期间尽可能不受其他事务影响 -> 读操作可能引起的问题 -> 事务的隔离级别

Durability：持久性，事务一旦提交对数据库的改变就应该是永久性的，不会因为宕机等原因造成数据丢失。实现基于redo log

读操作可能引起的问题

脏读：当前事务读取其他事务未提交的数据

不可重复读：事务前后读取同一个数据，得到的结果不同。读了未提交和已经提交的数据

幻读：事务先后两次按照同一条件查询所得到的结果行数不同

事务的隔离级别

隔离级别 脏读 不可重复度 幻读
读未提交 可能 可能 可能
读已提交 不可能 可能 可能
可重复读 不可能 不可能 可能
串行化 不可能 不可能 不可能

InnoDB 的默认隔离级别是 RR（可重复读），通过 MVCC（Multi-Version Concurrency Control） 和 next-key 锁解决幻读问题

InnoDB 与 MyISAM区别

InnoDB MyISAM
支持事务，支持外键 不支持事务，不支持外键
锁的最小粒度为行锁 锁的最小粒度为表锁
聚集索引，文件存放在主键索引的叶子节点上 非聚集索引，索引与数据分离，索引保存的是数据文件的指针

B树与B+树的区别

B树 B+树
子节点的个数是关键字个数+1 子节点的个数就是关键字节点的个数
叶子节点之间通过指针进行连接
非叶子节点和叶子节点都存储关键字和数据 只有叶子节点存储数据，查询时间复杂度固定为 O(logn)

为什么B+树比B树更适合数据库索引

  1. 相比B树B+树减少了IO读写的次数，由于非叶子节点只存储关键字，所以每一页可以存储更多的关键字，磁盘随机IO次数也就减少了
  2. B+树的查询比B树的查询更加稳定，由于数据都在叶子节点上，所以每次查询经过的路径长度都相同
  3. B+树的叶子节点通过指针连接，更加有利于有益于范围查询和有序遍历，只要遍历叶子节点即可，而B树要通过中序遍历得到有序的序列。

计算机网络

TCP三次握手

第一次握手：

建立连接时，客户端向服务器发送（SYN = 1）请求建立连接包（syn=x），进入SYN_SEND 状态，等待服务器确认。SYN = 同步序列编号

第二次握手：

服务器收到（SYN）请求建立连接报文，必须确认客户的SYN包（ack = x+1），向客户端发送ACK包，同时自己也发送一个SYN（syn=k）包，此时服务器进入SYN_RECV状态

第三次握手：

客户端收到服务器的SYN+ACK包，向服务器发送确认包ACK（ack=k+1），发送完毕后，客户端和服务器进入ESTABLISHED状态，此时TCP连接成功，三次握手完成

TCP 四次挥手

  1. 客户端发出连接释放报文，FIN = 1 seq = u，并且停止发送数据，客户端进入终止等待1阶段
  2. 服务器收到客户端发送的连接释放报文，发出确认报文（ACK=1, ack=u+1），并且带上自己的序列号(syn=v)，此时服务器端就进入了CLOSE_WAIT关闭等待阶段，服务器将最后的数据发出
  3. 客服端收到服务器发出的确认报文后，客户端就进入了终止等待2阶段等待服务器发送连接释放报文，在此之前客户端还需要接受服务器发送的最后的数据
  4. 服务器将最后的数据发送完后，向客户端发送连接释放报文，此时假定报文的序列号为w，服务器进入了LAST_ACK（最后确认）阶段，等待客户端的确认
  5. 客户端收到连接释放报文后，发出确认报文，ack=w+1，seq=u+1，此时客户端进入了 TIME_WAIT阶段，但TCP连接还未释放，必须经历两个MSL即最大报文寿命后，客户端才撤销相应的TCB，进入CLOSED状态
  6. 服务器收到了客户端发出的确认报文后，立刻进入CLOSED状态，之后服务器端撤销相关的TCB，结束这次TCP连接

TIMEWAIT 有什么用，为什么要等这个时间

我们必须假设网络是不可靠的，假设最后一个ACK报文丢失，服务器端没有收到ACK报文将重复发送FIN报文，假如客户端在TIMEWAIT这段时间内能够收到FIN报文，那么客户端重新发送ACK报文，MSL是最大报文生存时间，2MSL就是一个报文发送和一个回复所需的最大时间。

为什么连接的时候是3次握手，释放连接的时候是四次挥手

当服务器端收到客户端的申请连接报文后，可以直接发送SYN报文和ACK报文，同时进行同步和确认。但当释放连接时，服务器端收到释放连接的报文时，并不会直接关闭SOCKET，FIN报文和ACK报文不能同时发出，FIN代表没有数据要发送了。所以只能先发送一个确认报文给客户端，等待所有数据都发送完毕后，才向客户端发送连接释放报文，所以需要四次挥手

为什么不用两次握手建立连接（TCP握手为什么需要三次）

3次握手完成了两个重要的功能，一是双方做好发送数据的准备工作，二是双方就初始序列号进行协商。如果把3次握手改成两次握手，那么可能发生死锁，首先客户端向服务器端发送SYN报文，服务器端收到后向客户端发送SYN和ACK报文，但是ACK报文在传输的过程中可能丢失，此时服务器端认为连接已经建立，开始发送数据，但客户端没有收到ACK报文，认为连接还未建立，忽略服务器端发来的数据，只等待确认报文，而服务器端在发出的分组超时后，重复发出同样的数据，造成死锁。

如果已经建立了连接，但客户端出故障了怎么办

TCP设有一个保活计时器，时间通常设为2小时，客户端如果出现故障，服务器端也不会一直等待下去，如果在计时器内没有收到客户端的任何数据，服务器就会每隔75s发出一个探测报文段，如果发送了10个报文段客户端依旧没有发送任何数据，服务器就关闭连接。

TCP 与 UDP 的区别

TCP面向连接，UDP无连接

TCP是可靠传输，UDP是不可靠传输

UDP支持多对多相互通信，TCP只支持一对一通信

TCP 拥塞控制

慢启动

首先将 cwnd(拥塞窗口) 置1，每收到一个新的ACK报文段，将拥塞窗口加一，这样每经过一个RTT，拥塞窗口翻倍

拥塞避免

如果 拥塞窗口 超过了 慢启动阈值 那么就将线性增长，每个RTT加一

快重传

接收方每收到一个失序的报文后，立即发送重复确认报文

发送方收到三个重复确认报文就立即发送丢失的报文

快恢复

当发送方收到三个重复确认报文时，将 慢启动阈值设为当前cwnd大小的一半

拥塞窗口也设为阈值+3，继续回到拥塞避免阶段。

HTTP

HTTP与HTTPS有什么区别

HTTP传输的协议都是未加密的，所以非常不安全，https 是由 ssl 和http构建的可进行加密传输，身份认证的网络协议，要比http协议安全。http的默认端口为80，https协议的默认端口为443。

HTTPS加密方式

不对称加密和对称加密：

  • 服务器向浏览器发送自己的公钥
  • 浏览器随机生成一个用来加密的密钥并用收到的公钥进行加密
  • 将加密后的密钥发送给服务器，服务器用私钥解密，得到用来加密信息的密钥
  • 之后双方就使用此密钥加密要发送的信息

为了防止中间人劫持，需要有CA机构的数字证书和数字签名。浏览器保有CA机构的公钥，对数字签名进行解密，对明文T用指定的hash算法进行hash，得到的值应该与数字签名解密后得到的值相同。

HTTP 错误码

  • 1xx：消息类，表示服务器的临时回应
  • 2xx：表示浏览器端请求被成功处理
  • 3XX：重定向，301永久，302临时
  • 4XX：错误，403资源不可用，404无法找到指定位置的资源
  • 5XX：服务器错误，502 Bad GateAway， 503 Service unavailable

操作系统

内核

内核是操作系统最基本的部分，向外提供了计算机设备的核心管理调用.

用户态：大多数进程都运行在用户态下，此时进程能访问的内存空间和资源对象都受到限制，所占有的处理器是可以被抢占的

内核态：进程可以访问所有的内存和对象，占有的CPU不能被抢占。

用户态切换到内核态

内存模型

栈区：由操作系统分配和释放，存放局部变量和函数参数等，

堆区：由程序员申请和释放，非连续空间

常量区：存放常量

静态区：存放全局变量与静态变量

代码区：存放代码

段与页的区别

段是信息的逻辑单位，页是信息的物理单位。

页的大小不可改变，段的大小可以动态改变。

分页主要是用于实现虚拟内存，实现更大的地址空间。分段主要是为了使程序和数据可以划分为逻辑上独立的地址空间并且有助于共享和保护。

进程与线程

进程是资源分配的基本单位，线程是CPU调度的基本单位

一个进程可以拥有多个线程，线程没有地址空间，包含在进程的地址空间内。

同一进程中的线程拥有自己的栈和寄存器，共享数据段，代码段和文件

进程间的通信方式

  1. 低级通信方式：同步与互斥，PV操作为低级进程通信原语
  2. 高级通信方式：
    • 共享存储器系统
    • 消息传递系统：
      1. 直接通信：消息队列
      2. 间接通信：信箱通信
    • 管道通信系统或者共享文件系统

I/O多路复用 select poll epoll 的区别

selet 函数监视的文件分3类: writefds, readfds, exceptfds， select函数返回后，轮询所有fds，找到就绪的描述符

poll 使用一个 pollfd的指针，轮询pollfd来获取就绪的描述符

epoll

一旦基于某个文件描述符准备就绪时，内核会采用类似 call_back 的回调机制，迅速激活这个文件描述符，当进程调用 epoll_wait() 时便得到通知

IO效率不会随着fd的增加而增加
